myFirstPost

第一次使用hexo进行博客编写并进行测试

使用过程

  • 1 安装nodejs环境后，查看node版本
    • node -v
  • 安装cnpm
    • npm install cnpm -g –registry=https://registry.npm.taobao.org
  • 安装 Hexo
    • cnpm i -g hexo-cli
    • 查看版本：hexo -v
  • 创建本地博客
    • cd d:/hexo/blog
    • hexo init
    • cnpm i
    • hexo s // 打开环境
  • 创建新的文章
    • cd d:/hexo/blog
    • hexo new “My First Post”
  • 创建静态文件
    • hexo g
  • 打开博客
    • 浏览器输入localhost:4000 新文章已出现

部署过程

  • 打开_config.yml文件
    • #Site下
      • title: 博客名
      • subtitle: ‘’
      • description: ‘’ // 描述信息
      • keywords:
      • author: 你的姓名
      • language: zh-Hans // 默认是en
      • timezone: ‘’
    • #URL下
      • url: 网站名称
      • root： 仓库名
    • #Deployer下
      • type: git
      • repository: 仓库链接
      • branch：分支名称
  • 安装自动部署发布工具
    • cnpm i hexo-deployer-git -S
  • 发布
    • hexo clean && hexo g && hexo d
      • 首次发布需要输入账号和密码
  • 打开服务
    • 进入gitee blog仓库的页面->服务->Gitee Pages->启动

目录结构

  • scaffolds:模板、脚架目录
  • source：源文件夹(内容核心)，所有的文章和分类、标签等都是通过该文件夹下的内容进行发布的
    • /_post：所有发布的文章都在该文件夹中
  • themes：主题存放文件夹，把所需的主题目录添加到该文件夹中并更改项目_config.yml相应配置即可更换主题
  • _config.yml：项目配置文件，设置项目的通用配置(主题外的配置，如标题、分页、搜索、作者、发布地址等)
